// HOW TO READ THESE RESULTS (from the Titanic Streamlit app) // 1) Accuracy: overall fraction of correct predictions. High accuracy can be misleading on imbalanced classes. // 2) Precision: of the passengers predicted as 'survived', how many actually survived. Important when false positives are costly. // 3) Recall (Sensitivity): of all actual survivors, how many the model correctly identified. Important when missing positives is costly. // 4) F1 Score: harmonic mean of precision and recall ‚Äî use it when you need a balance between precision and recall. // 5) ROC AUC: area under ROC curve; shows separability between classes across thresholds (1.0 is perfect, 0.5 is random). // 6) Sample Size: number of test samples used to compute metrics; smaller samples increase variance in metrics. // // Confusion Matrix: rows = actual classes, columns = predicted classes. // - True Negatives (TN): correctly predicted 'No' (did not survive). // - False Positives (FP): predicted 'Yes' but actual 'No' ‚Äî overestimates survivors. // - False Negatives (FN): predicted 'No' but actual 'Yes' ‚Äî missed survivors (costly if we want to rescue). // - True Positives (TP): correctly predicted 'Yes' (survived). // Interpretation tip: look at FN vs FP tradeoff depending on the application goal. // // ROC Curve: plots TPR (recall) vs FPR for different thresholds. Use it to choose an operating threshold that balances recall and FPR. // AUC close to 1.0 indicates good separability; compare AUC across models to choose the best one. // // Feature Importance: indicates which input variables the RandomForest used most to split the data. // - Higher importance means the model relied more on that feature to make decisions. // - Use feature importance to guide further data collection or to interpret model behavior. // // Classification Report: per-class precision, recall, f1 and support. Check 'support' to see class imbalance influence. // // Practical notes: // - If recall is prioritized (e.g., catching true survivors), lower threshold to increase recall at cost of precision. // - If precision is prioritized (e.g., avoiding false survivor alerts), raise threshold to increase precision. // - Always examine confusion matrix and ROC together; numerical metrics alone can hide specific error patterns. // - Consider cross-validation and confidence intervals for more robust metric estimates. // - The visualizations below (confusion matrix, ROC curve, feature importance) are illustrative snapshots generated by the app. // // How this HTML page maps to the Streamlit app: // - Top metrics (Accuracy/Precision/Recall/F1/ROC AUC/Sample Size) replicate the values computed in the Streamlit 'M√©tricas do Modelo' tab. // - Confusion matrix and ROC plots are simplified static SVG representations here; the Streamlit app renders Matplotlib/Seaborn plots with exact numbers. // - Feature importance shown here is a compact bar-like representation; in the app, a horizontal bar chart is generated for better resolution. // - Classification report table here is an excerpt; consult the live app or saved metrics file to view the full report object (JSON/pickle). // // Notes for maintainers: // - Metrics are saved to 'titanic_metrics.pkl' by the app; to keep this page fully up-to-date, export metrics to JSON and load them here during build/deploy. // - If you want dynamic integration, add a small script to fetch a '/metrics.json' file produced by the app and inject values into the DOM.
